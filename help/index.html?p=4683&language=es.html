Problemas Generales

Al arrancar el Applet, no se reconoce el certificado de firma de código con el que está firmado, provocando una alerta de seguridad, ya que no se confía en la procedencia del creador.

Esto está provocado por el plugin de soporte Java en los navegadores (IcedTead), que sólo verifica 1 nivel de certificados antes de declararlo como de “no-confianza”. El código del Applet de Tractis está firmado por Thawte que usa una CA Intermediaria para la emisión de certificados de entidad final.

  • El problema está reportado en la web de IcedTea-Web sin resolver – checks certs only upto 1 level deep before declaring them untrusted

Tractis Contracts (firma electrónica con certificados)

En este momento existe un problema que hace que no se puedan firmar en Tractis Signatures (https://www.tractis.com/signatures). El problema se está investigando a fondo, y parece ser que es un problema derivado de la no-confianza por parte del plugin ﻿﻿﻿IcedTea-Web de certificados no emitidos por una Root CA, en el caso concreto de la firma, cuando se producen las verificaciones de seguridad e integridad entre el Applet y el servicio de Firmas.

Seguiremos trabajando para conseguir un perfecto funcionamiento en entornos Linux totalmente Libres con OpenJDK/IcedTea, de momento, y esperando que el problema de IcedTea se resuelva, recomendamos instalar Java 1.6 de SUN/Oracle, incluido en los repositorios Partner de las versiones recientes de Distribuciones populares como Ubuntu.
