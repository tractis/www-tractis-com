¿Cual es la diferencia entre la Verificación de identidad vía Pasarela Front-end y vía Pasarela Back-end?

Puedes integrar Tractis Identity Verifications en tu sitio web o aplicación de dos maneras:

  1. Verificación de identidad vía Pasarela Front-end.
  2. Verificación de identidad vía Pasarela Back-end.

A continuación, se describen las principales diferencias entre ambas opciones:

1. Verificación de identidad vía Pasarela Front-end

  • Descripción: En el momento de realizar la verificación de identidad, el usuario abandona tu sitio web y es redirigido a la pasarela de verificación de identidad de Tractis. Una vez completado el proceso de verificación de identidad, el usuario es redirigido de vuelta a tu sitio web. En otras palabras, el proceso es muy similar a ser redirigido a una pasarela de pagos de un banco. La única diferencia es que, en vez de ser la pasarela de pagos de un banco, en este caso se trata de la pasarela de verificación de identidad de Tractis.
  • Coste: 0,05 € por verificación en la Tarifa Prepago de Tractis.
  • Apariencia: Puedes personalizar la pasarela con tu logo y colores corporativos. Con o sin personalización, la pasarela de Tractis mostrará una URL “tractis.com” y Tractis se reserva la posibilidad de mostrar un logo de “Powered by Tractis” y de ofrecer al usuario la posibilidad de adquirir un lector de tarjetas inteligentes.
  • Tiempo estimado de puesta en producción: 3 horas.
  • Dificultad de puesta en producción: Fácil.
  • Desarrollos necesarios: Para utilizar esta opción, deberás seguir los siguientes pasos:
    1. Obtener una API Key de Tractis Verifications.
    2. Crear un formulario HTML para acceder a la pasarela y verificar los datos recogidos por la pasarela siguiendo estas instrucciones.

2. Verificación de identidad vía Pasarela Back-end

  • Descripción: El usuario nunca abandona tu sitio web para verificar su identidad.
  • Coste: 0,05 € por verificación en la Tarifa Prepago de Tractis.
  • Apariencia: Dado que el usuario nunca abandona tu sitio web, tienes control total sobre la interface (logos, colores, marca, copy, etc) que el usuario verá durante el proceso de verificación de identidad. El usuario no es consciente en ningún momento de que la verificación de su certificado está siendo realizada por Tractis.
  • Tiempo estimado de puesta en producción: 1 mes.
  • Dificultad de puesta en producción: Difícil. Requiere conocimientos sobre el funcionamiento y manipulación de certificados electrónicos.
  • Desarrollos necesarios: Para utilizar esta opción, deberás seguir los siguientes pasos:
    1. Obtener una API Key de Tractis Verifications.
    2. Implementar la extracción del certificado del usuario, lo que conlleva implementar los métodos de acceso y extracción de los certificados. Para ello, puedes utilizar un módulo específico desarrollado por Tractis para el servidor web Apache o realizar un desarrollo ad-hoc de integración.
    3. Invocar la API de Tractis Identity. Para ello, deberás autenticarte con tu usuario de Tractis y proporcionando la API Key utilizada y el certificado extraído del cliente.

Nota importante:

De manera general, si no estás familiarizado con conceptos como “comandos APDU”, “pkcs12″, “pkcs11″ o “cryptoAPI en Windows”, te recomendamos encarecidamente utilizar la Verificación de identidad via Pasarela Front-end.
